Macro Library graphtheory

A library of graph theory and scheduling functions. Version 1.1, April 3, 2012

Most graphing functions in this library use an options array. Here are the common options - specific functions will mention other options.

  • options['width'] = width of output, in pixels. Defaults to 300.
  • options['height'] = height of output, in pixels. Defaults to 300.
  • options['digraph'] = true/false. If true, g[i][j] > 0 means i leads to j.
  • options['arrowoffset'] = position (0-1) along edge where digraph arrow should go.
  • options['useweights'] = true/false. If true, g[i][j] used as weight.
  • options['labels'] = "letters" or array of labels. If "letters", letters A-Z used for labels. If array, label[i] used for vertex g[i].
  • options['weightoffset'] = position (0-1) along edge where weights should go.
  • options['connected'] = true/false. When randomizing graphs, whether you want to force the result to be connected. If false for a tree, this forces a disconnected graph.
  • options['randweights'] = max or array(min,max). Randomizes weights of edges.
  • options['randedges'] = probability (0-1). Randomly keeps edges of original graph with given probability.
  • options['colors'] = array of colors for vertices and labels.
  • options['tree'] = true. Creates a minimum cost spanning tree from the original graph.
  • options['labelposition'] = "above","below","right","left","aboveright",etc. Position of vertex labels.

graphcircleladder

graphcircleladder(n, m, [options])

Draws a circular ladder graph, with n vertices around a circle and m concentric circles, connected around the circle and between circles.

Returns array(pic,g), where pic is the image and g is the graph object you may need for other functions.

Examples
$pic,$g = graphcircleladder(6, 2)

graphnestedpolygons

graphnestedpolygons(n, m, [options])

Draws a graph of offset nested polygons, with n vertices around a polygon and m concentric polygons. Vertices of the inner polygon touch the midpoint of the outer polygon.

Returns array(pic,g), where pic is the image and g is the graph object you may need for other functions.

Examples
$pic,$g = graphnestedpolygons(5, 2)

graphcircle

graphcircle(n, [options])

graphcircledstar

graphcircledstar(n, [options])

Returns array(pic,g), where pic is the image and g is the graph object you may need for other functions.

Examples
$pic,$g = graphcircledstar(6)

graphbipartite

graphbipartite(n, m, [options])

Draws a complete bipartite graph (every vertex on the left is connected to every vertex on the right), with n vertices in the first column, m in the second.

Returns array(pic,g), where pic is the image and g is the graph object you may need for other functions.

Examples
$pic,$g = graphbipartite(3, 4)

graphdijkstra

graphdijkstra(g, [dest])

Computes Dijkstra's algorithm on the graph g. g is a 2-dimensional matrix; g[i][j] > 0 if vertices i and j are connected. If dest vertex is not set then the last vertex will be used as the destination vertex.

Returns array(dist,next) where for each vertex i, dist[i] is the shortest distance to end, and next[i] is the vertex next closest to the end.

Examples
$dist, $next = graphdijkstra($g)

graphkruskal

Examples
$mst = graphkruskal($g)

graphnearestneighbor

graphnearestneighbor(g, start, [getweight])

Returns a Hamiltonian circuit graph using nearest neighbor starting at vertex start. If getweight is set to true, this will return array(circuit,total weight of circuit).

Examples 
$circuitg = graphnearestneighbor($g, 0)

graphrepeatednearestneighbor

graphrepeatednearestneighbor(g, [multi])

Returns an array (graph,mincostarr), where graph is the Hamiltonian circuit graph using repeated nearest neighbor, and mincostarr is an array of starting vertex indices that gave circuits with the same minimum cost. If multi is set to true, an array of graphs is returned rather than just one.

Examples 
$circuitg, $starts = graphrepeatednearestneighbor($g)

graphsequenceeuleredgedups

graphsequenceeuleredgedups(g, op, seq)

Determines if the given sequence of labels determines an edge-covering circuit on graph g. Options op is needed for labels.

Returns -1 if not all edges covered or seq uses nonexisting edges. Returns 0 if all edges covered with no duplication. Returns the number of duplications if it covers all edges with duplications.

Examples
$dups = graphsequenceeuleredgedups($g, array('labels'=>'letters'), "ABCDA")

graphshortestpath

graphshortestpath(g, op, start, end, [type])

Finds the shortest path on graph g from vertex start to end.

Returns array(shortest path,length of path). For path, type=0 returns a labeled path, like "ABCD"; type=1 returns an array of vertex indices.

Examples
$path, $len = graphshortestpath($g, array('labels'=>'letters'), 0, 3)

graphcomparecircuits

graphcomparecircuits(A, B)

Compares two circuits to see if they are the same, regardless of starting vertex. So "ABCDA" would be considered equivalent to "DCBAD". Can be used with the conditional answer type to score circuits.

Returns true or false.

Examples
graphcomparecircuits("ABCDA", "DCBAD")  // true

graphgridlayout

graphgridlayout(graph, [options])

Draws a graph based on a graph adjacency matrix using a rectangular grid layout. Could hide some edges that connect colinear vertices. Use options['wiggle'] = true to perturb off the exact grid. g is a 2-dimensional matrix; g[i][j] > 0 if vertices i and j are connected.

Examples

graphrandomgridschedule

graphrandomgridschedule(n, m, p, [options])

Draws an n by m grid of vertices. Each pair of neighboring and diagonal vertices has a p probability (0 to 1) of being connected. An end vertex is added.

  • options['weights'] as an array of n*m elements will be used as weights.
  • options['weights'] as a single number will randomize weights from 1 to that value.
  • If options['labels'] are used, "start" and "end" will be added automatically.
Examples
$pic,$g = graphrandomgridschedule(3, 3, 0.6, array('weights'=>10))

graphbackflow

graphbackflow(g, [w])

Computes the longest-path algorithm on the graph g. g is a 2-dimensional matrix; g[i][j] > 1 if vertex i leads to j. This might give bad/weird results if the graph has a circuit. The last vertex will be used as the destination vertex. w are weights for the vertices, if a scheduling digraph and tasks rather than edges have weights.

Returns array(dist,next) where dist[i] is the longest distance to end, and next[i] is the vertex next closest to the end.

Examples
$dist, $next = graphbackflow($g, array(5,2,8,3))

graphlistprocessing

graphlistprocessing(g, t, L, p, [options])

Calculates the list processing algorithm on p processors with priority list L. t is an array of task times. L is an array of indices into g showing the priority list. g is a 2-dimensional matrix; g[i][j] > 0 if vertex i leads to j. This might give bad/weird results if the graph has a circuit. The last vertex will be used as the destination vertex; it should be the last task on the priority list and have task time 0.

Output is out[processor] = array of array(task, timestarted, tasklength).

Examples
$schedule = graphlistprocessing($g, array(5,2,8,3,0), array(2,0,1,3,4), 2)
