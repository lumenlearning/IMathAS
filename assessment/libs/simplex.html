simplex

simplex(type, objective, constraints)

Creates and returns a new simplex matrix. Elements are fractions stored in the form of an array(numerator, denominator).

  • type: a string that contains either "max" or "min".
  • objective: list or array of the coefficients.
  • constraints: an array that contains the inequality information. Constraints are composed of three parts: first part is a list or array of the coefficients in the inequality; second part is the inequality '<=' or '>='; third part is the right hand number.

Use simplexdisplaytable() to create a string that can be used for display.

Examples
// objective function: f = x1+7x2+5x3
$objective = array(1,7,5)
// or
$objective = "1,7,5"

// constraint inequality: 3x1+x3 <= 35
$constraints[0] = array(array(3,1,0), "<=", 35)
// or
$constraints[0] = array("3,1,0", "<=", 35)
// first part: array(3,0,1) or "3,0,1"; second part: "<="; third part: 35

$sm = simplex("max", $objective, $constraints)

simplexchecksolution

simplexchecksolution(type, HasObjective, solutionlist, stuanswer, [ismixed=FALSE])

  • type: a string that contains either "max" or "min".
  • HasObjective: either 0 (default, objective value is not included in the stuanswer array) or 1 (objective value is included and is the last column in the stuanswer array).
  • solutionlist: an array of solutions (in the case of multiple solutions), in the form solutionlist[0] = array(solution values for matrix[0], IsOptimized), solutionlist[1] = array(solution values for matrix[1], IsOptimized), etc. This is returned from simplexsolve.
  • ismixed: an optional flag for the function that tells the routine to read max values instead of min ones for mixed constraints. Default is FALSE.
  • stuanswer: the answer the student submitted.

Returns: 0 if no match is found, 1 if a match is found.

Examples
$match = simplexchecksolution("max", 1, $solutionlist, $stuanswers[$thisq])

simplexcreateanswerboxentrytable

simplexcreateanswerboxentrytable(rows, cols, [startnumber, matrixname, linemode, header, tablestyle])

Creates a string that is a valid HTML table syntax for displaying answerboxes.

  • rows: number of rows to make.
  • cols: number of columns to make.
  • startnumber: the starting number for the answerbox. Default is 0.
  • matrixname: a string that holds the matrix name, like A or B. This does not contain tick marks or brackets - if you want them you need to supply them. Default empty string.
  • linemode: show none, augmented, or simplex, value is either 0, 1 or 2. 0 shows no lines; 1 shows an augmented line; default 2 shows simplex lines.
  • header: list or array of the variables "x1,x2,x3" that are used for the column titles. Default none.
  • tablestyle: any additional styles for the table that you may want, like "color:#40B3DF;". Default none.

Returns: valid HTML table syntax for displaying answerboxes.

Examples
$t = simplexcreateanswerboxentrytable(3, 5, 0, "", 2, "x1,x2,x3,s1,s2")

simplexcreateinequalities

simplexcreateinequalities(type, objectivevariable, objective, constraints, [headers, displayASCIIticks, showfractions, includeinequalities])

Creates an array of strings that correspond to each line of the simplex inequalities.

  • type: a string that contains either "max" or "min".
  • objectivevariable: the name of the objective function, like f or g.
  • objective: list or array of the coefficients.
  • constraints: an array that contains the inequality information, in the form array(array(3,1,0),"<=",35): first part is a list or array of the coefficients in the inequality; second part is the inequality (<= or >=); third part is the number on the other side of the inequality.
  • headers: list or array of the variable names to use. Default "x1,x2,x3, ..." for max, "y1,y2,y3, ..." for min.
  • displayASCIIticks: put tick marks around each element of the table, either 0 or 1. 0 does not use math ticks. Default 1 uses math ticks. Use 0 if you are building an answerbox matrix.
  • showfractions: either 0 (shows decimals) or 1 (default, shows fractions).
  • includeinequalities: either 0 (does not append the inequality and right hand side number, e.g. "<=",35) or 1 (default, includes the full inequality).

Returns: an array of strings.

Examples
$ineqs = simplexcreateinequalities("max", "f", $objective, $constraints)

simplexfindpivotpoint

simplexfindpivotpoint(simplexmatrix)

  • simplexmatrix: a valid simplex matrix.

Returns array(condition, pivotpoints) where:

  • condition: -1 means No Solution; 0 found pivot point(s); 1 means no pivot points found; 2 found possible multiple solution pivot point(s).

simplexreadsolution

simplexreadsolution(simplexmatrix, type, showfractions)

Reads the simplex matrix to find the current solution to the optimization problem. Returns an array that contains the solution.

  • simplexmatrix: a valid simplex matrix.
  • type: a string that contains either "max" or "min".
  • showfractions: either 0 (shows decimals) or 1 (default, shows fractions).

Returns: array(solution values for sm, IsOptimized).

For Max solutions the solution is an array in the form of: x1, x2, etc, s1, s2, etc., f, IsOptimized where f contains the maximum value and IsOptimized contains either a Yes or a No (objective has been reached).

For Min solutions the solution is an array in the form of: x1, x2, etc, y1, y2, etc., g, IsOptimized where g contains the minimum value and IsOptimized contains either a Yes or a No (objective has been reached).

Examples
$solution = simplexreadsolution($sm, "max", 1)

simplexsolve2

simplexsolve2(simplexmatrix, type, [showfractions=1])

Mixed constraints. This method solves the minimization problem which has the following conditions:

  1. The objective function is to be minimized.
  2. All variables are nonnegative.
  3. The constraints are of the form: a1x1+ a2x2+ ... + anxn >= b where b is any constant.
  • simplexmatrix: a valid simplex matrix.
  • type: a string that contains either "max" or "min".
  • showfractions: either 0 (shows decimals) or 1 (default, shows fractions).

Returns: (simplexsets[][], $objectivereachedsolutionlist, runtime)

  • simplexsets[][] = a 2 dimensional array used to build an output for the various paths that could be used to solve the simplex, where columns are: parent column number (column of the parent simplex, zero based), pivot point condition, pivot point (point that will be pivoted on), pivot points (array of all possible pivots), simplexmatrix (simplex matrix to pivot on), solution (the solution to the simplex matrix).
Examples
$simplexsets, $reachedlist, $runtime = simplexsolve2($sm, "min")

simplexsolve

***** DEPRECATED - use simplexsolve2 *****

simplexsolve(simplexmatrix, type)

This method solves the standard maximization problem which has the following conditions:

  1. The objective function is to be maximized.
  2. All variables are nonnegative.
  3. The constraints are of the form: a1x1+ a2x2+ ... + anxn <= b where b > 0.
  • simplexmatrix: a valid simplex matrix.
  • type: a string that contains either "max" or "min".

Returns: array(solutionlist, smlist, pivotlist)

  • solutionlist: an array of solutions (in the case of multiple solutions), in the form solutionlist[0] = array(solution values for matrix[0], IsOptimized), solutionlist[1] = array(solution values for matrix[1], IsOptimized), etc.
  • smlist: an array of simplex matrices, with sma[0] = initial simplex matrix, sma[1] = the result of the first pivot, sma[2] = the result of the second pivot, sma[3] = the result of the third pivot, etc.
  • pivotlist: an array of pivot points, with pivotlist[0][0] = the pivot used for the first pivot, pivotlist[0][1] = if exists, another possible pivot point for this step, pivotlist[1][0] = the pivot used for the second pivot, pivotlist[2][0] = the pivot used for the third pivot, etc.
Examples
$solutionlist, $smlist, $pivotlist = simplexsolve($sm, "max")
