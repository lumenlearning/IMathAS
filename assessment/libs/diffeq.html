diffeq_slopefield

diffeq_slopefield(function, [options])

Draws a slopefield for the given differential equation.

function: For a first order ODE, like dy/dx=x*y^2, use a single string, like "x*y^2". For an autonomous system, like x'=x+y, y'=y-2x, list both parts, like "x+y,y-2x"

options is an optional associative array of parameters. You can define none or several of them. The options you can set are:

  • dx: x spacing of slopes. default 1
  • dy: y spacing of slopes. defaults to match dx
  • vars: The variables of the diffeq. default "x,y"
  • arrows: Set to 1 to show arrows on slopes. default 0
  • format: overall format. default "slopefield". Set to "vectorfield" for vectorfields which will set norm to false and arrows to 1, and output vectors in the accessible alternative.
  • xmin: default -4
  • xmax: default 4
  • ymin: default -4
  • ymax: default 4
  • xgrid: x spacing of grids. defaults to 0 (off)
  • ygrid: y spacing of grids. defaults to match xgrid
  • xlbl: x spacing of ticks/labels. defaults to 1
  • ylbl: y spacing of ticks/labels. defaults to match xlbl
  • labels: set to 0 to hide axis labels. default 1
  • linewidth: width of slope lines. default 1
  • color: color of lines. default blue
  • length: length of slope lines, as a fraction of dx. default 0.75
  • norm: whether to normalize slopes to the same length. default true
  • caption: caption for the accessible table. defaults to something appropriate to format.
  • width: drawing width. default 300
  • height: drawing height. default 300
Examples
// using defaults 
$g = diffeq_slopefield("x*y^2")

// setting options inline
$g2 = diffeq_slopefield("xy", ['dx'=>0.5, 'arrows'=>1])

// or building the options array separately
$ops['dx'] = 0.5
$ops['arrows'] = 1
$g3 = diffeq_slopefield("xy", $ops)

// system x'=x+y, y'=y-2x, as vector field
$g = diffeq_slopefield("x+y,y-2x", ['format'=>'vectorfield'])

diffeq_comparegensoln

diffeq_comparegensoln(expr1, expr2, vars, constants, [tolerance, domain])

Compares two general solutions to an ODE which contain arbitrary constants. They are compared in a way that allows swapping which constant goes where, and accommodates absorbed constants.

  • expr1: the first general solution, like "cos(t)+c_1*e^t+c_2*e^(-t)"
  • expr2: the second general solution
  • vars: the independent variables in the expression, like "t"
  • constants: the arbitrary constants in the expression, like "c_1,c_2"
  • tolerance: the relative tolerance to compare with. Default 0.001.
  • domain: the domain to compare on. Default "-10,10".
Examples
$answer = "cos(t)+c_1*e^t+c_2*e^(-t)"
if (stuansready($stuanswers,$thisq)) {
  $stu = $stuanswers[$thisq]
  if (diffeq_comparegensoln($correct, $stu, "t", "c_1,c_2")) {
    $answer = $stu    // set stuans as answer if equivalent
  }
}
