Macro Library logic

Version 1.0: April 18, 2022

Created by Avery Flowers

A collection of functions designed to aid in the creation of basic logic questions.

Note: All variables used in these functions must be single alphabetical characters. Greek words or symbols are not supported.

Note: The valid logical operations are 'and', 'or', 'xor', 'neg', 'implies' and 'iff'. Other synonymous words (such as 'not', 'ifthen', 'oplus', 'wedge', 'vee') can be used as well.

logicallyequivalent

logicallyequivalent(exp1, exp2, vars)

  • exp1: A logical expression.
  • exp2: A logical expression.
  • vars: An array/list of variables.

Returns: 1 (TRUE) if exp1 is logically equivalent to exp2. 0 (FALSE) otherwise.

Examples 
logicallyequivalent("not(p and q)","not p or not q","p,q") // 1 (true)

logicallyimplies

logicallyimplies(exp1, exp2, vars)

  • exp1: A logical expression.
  • exp2: A logical expression.
  • vars: An array/list of variables.

Returns: 1 (TRUE) if exp1 logically implies exp2. 0 (FALSE) otherwise.

Examples 
logicallyimplies("p and q","p","p,q")  // 1 (true)

logicevaluate

logicevaluate(exp, vars, truths)

  • exp: A logical expression.
  • vars: An array (or list) of variables contained within the expression.
  • truths: An array (or list) the same size as vars, containing binary or boolean values to be assigned to each entry in vars.

Returns: a binary digit corresponding to the truth value of exp if each variable in vars is replaced with the corresponding binary digit (truth value) in truths.

Examples
logicevaluate('a implies b', 'a,b', '1,0')
// 0 (FALSE), since a->b is false when a=1 and b=0

logicevaluateall

logicevaluateall(exp, vars)

  • exp: A logical expression.
  • vars: An array (or list) of variables contained within the expression.

Returns: a bitstring of length 2^|vars|. The bit in position i (indexed at 0) is the result of the logicevaluate function with the truth array equalling the binary representation of i.

Examples
logicevaluateall('a and b', 'a,b')
// array(0,0,0,1)
// The entry at index 2 (binary 10) corresponds to evaluating 'a and b' with a=1 and b=0.
// Remember that arrays are 0-indexed.

logicrand

logicrand(vars, ops, bnum, unum)

  • vars: An array (or list) of variables.
  • ops: An array (or list) of logical binary operations (and,or,xor,implies,iff).
  • bnum: An array (or list) of integers between 1 and 4.
  • unum: An array (or list) of integers between 0 and 2*(the minimum entry in bnum)+1.

Returns: a random logical expression containing variables from the vars array, operations from the ops array. The number of binary operations will be a random element of bnum. The number of negations will be a random element of unum. Tries to avoid 'silly' expressions such as (p and q) or (q and p) for example.

Examples
logicrand("p,q,r", "and,or,xor", 2, 1)
// a random expression using p, q, r with 2 binary operations (and, or, xor only)
// and 1 negation symbol, e.g. 'r and not (p xor q)'

logicmakepretty

logicmakepretty(exp)

  • exp: A logical expression.

Returns: converts the output of the logical functions here to a format to be 'prettified' (removing unnecessary parentheses, double negations, etc.) for display in the Question Text.

logicsteps

logicsteps(exp)

  • exp: A logical expression.

Returns: an array of logical subexpressions of exp, each entry applying one operation to the previous entry until the last entry is exp. This shows the 'steps' required to evaluate or construct exp.

Examples
logicsteps('(p and q) or not p')
// array('p and q', 'not p', '(p and q) or not p')

truthtable

truthtable(exp, [showResult=FALSE], [showSteps=TRUE], [offset=0])

  • exp: A logical expression.
  • showResult: If FALSE (default), each entry in the constructed truth table (beyond the atomic variables) will contain an answerbox [AB#]. If TRUE, the constructed truth table will be completed with T's and F's. If 2, the truth table will be filled in, and [AB#] will be placed in the header for the expression(s). If 3, the truth table will be filled in, and a single [AB] will be placed in the header for the expression.
  • showSteps: If TRUE (default), the constructed truth table will have a column for each subexpression of exp. If FALSE only the atomic variables and exp will be included in the table.
  • offset: Only applies when showResult is FALSE. Gives the index of the first answerbox to be contained in the truth table. For instance, if offset=3, then the first entry in the truth table will be '[AB3]'.

Returns: a string containing HTML code for a truth table for exp, using T's and F's for true and false.

Note: This does not change the $anstypes or $answer values. It should be used with the truthtableanswers function below.

Examples
truthtable('(p and q) or not p', TRUE, TRUE)
// returns a string for the table shown below (but in math display mode):
Truth table
pq(p and q)neg p(p and q) or neg p
TTTFT
TFFFF
FTFTT
FFFTT

truthtableanswers

truthtableanswers(exp, [showSteps=TRUE])

  • exp: A logical expression.
  • showSteps: If TRUE (default), the constructed truth table will have a column for each subexpression of exp. If FALSE only the atomic variables and exp will be included in the table.

Returns: an array of binary truth values containing the answer to each answerbox in the truthtable function.

Note: This is useful for autogenerating answers to the answerboxes found in truthtable. It does not affect $anstypes or $answer. It is recommended you place something like the following in the Common Control box:

Examples
$exp = 'p and q or not r'
$table = truthtable($exp, FALSE,TRUE)
$tableanswers = truthtableanswers($exp,TRUE)
$numboxes = count($tableanswers)-1
for($i=0..$numboxes){
    $anstypes[$i] = "choices"
    $displayformat[$i] = "select"
    $questions[$i] = array('F','T')  // Order is important: F=0 and T=1
    $noshuffle[$i] = "all"
    $answer[$i] = $tableanswers[$i]
}

logiclegend

logiclegend()

Returns: a simple string of HTML syntax that generates a table showing keyboard shortcuts for entering logic expressions.
